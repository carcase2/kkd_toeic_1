# 사용자 인증 시스템 구현 계획

## 개요
여러 사람이 각자의 단어와 학습 기록을 관리할 수 있도록 사용자별 데이터 분리 시스템을 구현합니다.

## 구현 방법

### 옵션 1: Supabase Auth 사용 (권장)
- **장점**: 
  - Supabase에 이미 통합되어 있음
  - 이메일/비밀번호, 소셜 로그인(Google, GitHub 등) 지원
  - 자동으로 사용자 세션 관리
  - RLS(Row Level Security)로 자동 데이터 분리
- **단점**: 
  - 이메일 인증 필요
  - 약간의 설정 필요

### 옵션 2: 간단한 닉네임 기반 (비인증)
- **장점**: 
  - 빠른 구현
  - 이메일 불필요
- **단점**: 
  - 보안이 약함
  - 데이터 보호 없음
  - 다른 사람이 같은 닉네임 사용 가능

## 권장: Supabase Auth 구현

### 필요한 작업

1. **데이터베이스 마이그레이션**
   - `words` 테이블에 `user_id` 필드 추가
   - `study_records` 테이블에 `user_id` 필드 추가
   - RLS 정책 설정

2. **인증 페이지**
   - 로그인 페이지
   - 회원가입 페이지
   - 로그아웃 기능

3. **API 수정**
   - 모든 API에서 현재 사용자 ID로 필터링
   - 사용자별 데이터만 조회/수정

4. **UI 수정**
   - 로그인 상태 표시
   - 로그아웃 버튼
   - 로그인하지 않은 경우 로그인 페이지로 리다이렉트
